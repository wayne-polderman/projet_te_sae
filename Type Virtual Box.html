Type Virtual Box

Type de virtualisation pour VirtualBox, VMware Workstation et Stratus

VirtualBox et VMware Workstation sont des environnements de virtualisation reposant sur un hyperviseur de type 2. Il s’agit d’une virtualisation hébergée, c’est-à-dire que l’hyperviseur s’exécute au sein d’un système d’exploitation hôte.

Il existe principalement deux types de virtualisation. La virtualisation avec un hyperviseur de type 1, dite bare metal, correspond à un hyperviseur installé directement sur le matériel d’une machine physique. Ce type d’hyperviseur offre de très bonnes performances, car il n’y a pas de système d’exploitation intermédiaire.

À l’inverse, la virtualisation avec un hyperviseur de type 2 repose sur un logiciel qui s’exécute à l’intérieur d’un système d’exploitation hôte. Ce type d’hyperviseur est généralement moins performant que le type 1, en raison de la surcouche logicielle, mais il est plus simple à mettre en œuvre et ne nécessite pas de matériel spécifique ni de compétences avancées.

Concernant Stratus, l’entreprise a développé la plateforme ztC Edge, qui propose une virtualisation intégrée permettant de consolider plusieurs applications sur une seule plateforme. Cette solution repose sur un hyperviseur basé sur l’architecture KVM (Kernel-based Virtual Machine).

KVM est un hyperviseur de type 1 (bare metal) pour Linux. Il permet d’exécuter directement le code machine sur le processeur hôte afin d’améliorer les performances. Le type de virtualisation proposé par Stratus est donc différent de celui de VirtualBox ou de VMware Workstation.
